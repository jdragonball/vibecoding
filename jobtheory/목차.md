# 나 사용설명서 - 목차 설계

## CSV 분석 인사이트

### 고민 패턴 (300건 분석)

| 패턴 | 빈도 | 핵심 Job |
|-----|------|---------|
| "~해도 될까요?" | 매우 높음 | 결정에 대한 **허락/확신** |
| "언제쯤" | 높음 | **타이밍** 알고 싶음 |
| "나한테 맞는" | 높음 | **적합성** 확인 |
| "잘 될까요?" | 중간 | **미래 예측** |
| 복합 고민 (연애+직업) | 높음 | 인생 전반의 **방향성** |

### 실제 고민 예시 (유료 고객 👑)

```
"이직하고 일이 잘풀리는거 같은대 뭔가 깊이는 없고
하루하루 쳐내는 느낌인데 이게 맞는가"
→ Job: 지금 길이 맞는지 확신받고 싶음

"진로를 고민하고 있어요. 연구원인데 박사학위 취득을
해야할 지 한다면 학계에 남을 지 회사로 가야할 지"
→ Job: A vs B 선택 도움

"성격때문에 죽을때까지 혼자살거 같은데 더 살아야하나"
→ Job: 존재 자체에 대한 위로와 희망
```

## 목차 구조 설계 (반동적 Semi-dynamic)

### 원칙
1. **큰 틀 10개 고정** → 일관성 유지
2. **세부 제목은 Claude가 고민에 맞게 생성** → 맞춤감
3. **3단계 구조**: 나를 알기 → 고민 분석 → 앞으로의 방향

---

### 고정 프레임워크 (10개 섹션)

```
[Part 1: 나를 알기]
1. 인트로 - "{이름}님은 이런 사람이에요"
2. 본질 - 사주로 본 나의 기본 에너지
3. 표현 - MBTI로 본 나의 행동 방식
4. 강점 - 타고난 재능과 잘하는 것
5. 약점 - 주의해야 할 부분

[Part 2: 고민 분석]
6. 고민의 뿌리 - {동적: 고민 맥락에 맞는 제목}
7. 지금 이 시기 - {동적: 현재 운의 흐름}
8. 핵심 조언 - {동적: 고민에 대한 직접적 답변}

[Part 3: 앞으로]
9. 구체적 행동 - {동적: 실천 가이드}
10. 마무리 - "{이름}님에게 전하는 말"
```

---

### 동적 제목 생성 규칙

**섹션 6 (고민의 뿌리)** - 고민 유형별 예시:
- 이직 고민 → "왜 지금 떠나고 싶을까요"
- 연애 고민 → "사랑이 어려운 이유"
- 금전 고민 → "돈이 안 모이는 패턴"
- 방향성 고민 → "길을 잃은 듯한 이 느낌"

**섹션 7 (지금 이 시기)** - 예시:
- "2026년, 움직여야 할 때"
- "잠시 멈춰도 괜찮은 시기"
- "씨앗을 뿌리는 해"

**섹션 8 (핵심 조언)** - 고민에 직접 답변:
- "이직해도 될까요?" → "지금 움직여도 됩니다"
- "연애 언제쯤?" → "올해 하반기를 주목하세요"
- "사업 해도 될까?" → "아직은 준비가 필요해요"

**섹션 9 (구체적 행동)** - 예시:
- "3개월 안에 해볼 것들"
- "관계를 바꾸는 작은 습관"
- "돈을 모으는 당신만의 방법"

## 프롬프트 전략

Claude에게 전달할 정보:
1. 사주 정보 (일간, 오행 분포)
2. MBTI
3. 고민 원문
4. 고정 프레임워크 (10개 슬롯)
5. 동적 제목 생성 지시

출력 요청:
```json
{
  "sections": [
    {
      "slot": 1,
      "fixedTitle": "{이름}님은 이런 사람이에요",
      "content": "..."
    },
    {
      "slot": 6,
      "dynamicTitle": "왜 지금 떠나고 싶을까요",  // Claude가 생성
      "content": "..."
    }
  ]
}
```

## 왜 이 구조인가?

### 완전 고정 vs 완전 동적 vs 반동적

| 방식 | 장점 | 단점 |
|-----|------|------|
| 완전 고정 | 일관성, 브랜딩 | 맞춤감 부족 |
| 완전 동적 | 맞춤감 극대화 | 일관성 없음, 품질 편차 |
| **반동적** | 일관성 + 맞춤감 균형 | 구현 복잡도 중간 |

### 서비스 관점

1. **공유 가능** - "너도 해봐" 했을 때 비슷한 구조라 비교 가능
2. **기대 관리** - "이 서비스는 이런 순서로 알려준다"
3. **품질 보장** - 최소한 10개 섹션은 보장
